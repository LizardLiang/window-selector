# 測試指南 - Window Selector

## 🔧 修正內容

### 問題 1: Win+B 被系統佔用 ✅ 已修正
- **原快捷鍵**: `Win+B` (被 Windows 系統佔用 - 聚焦到系統匣)
- **新快捷鍵**: `Ctrl+Alt+Q` (不衝突)

### 問題 2: 標籤模式按字母不跳轉 ✅ 已修正
- **原行為**: 按字母只選擇視窗，需要再按 Enter 才跳轉
- **新行為**: 按字母**直接跳轉**到視窗（類似 FastWindowSwitcher）

## 🎯 當前快捷鍵

| 功能 | 快捷鍵 | 行為 |
|------|--------|------|
| 全螢幕覆蓋層 | `Ctrl+Alt+Q` | 顯示縮圖方格 |
| 標籤模式 | `Win+Y` | 顯示視窗上方標籤 |

## 📋 測試步驟

### 測試 1: 全螢幕覆蓋層 (Ctrl+Alt+Q)

1. **準備**:
   - 開啟 3-5 個視窗（瀏覽器、記事本、檔案總管等）
   - 確保有些視窗被遮擋或最小化

2. **測試**:
   ```
   步驟 1: 按 Ctrl+Alt+Q
   預期: 看到全螢幕深色覆蓋層，顯示所有視窗的縮圖方格
   
   步驟 2: 按字母鍵（例如 S）
   預期: 對應的縮圖被高亮選中（系統強調色邊框 + 光暈）
   
   步驟 3: 按 Enter 或 Space
   預期: 跳轉到選中的視窗，覆蓋層消失
   
   步驟 4: 再次按 Ctrl+Alt+Q，按 Esc
   預期: 覆蓋層消失，回到原視窗
   ```

3. **進階測試 - 數字標籤**:
   ```
   步驟 1: 按 Ctrl+Alt+Q
   步驟 2: 按字母選擇視窗（例如 D）
   步驟 3: 按 Ctrl+1
   預期: 該視窗被標記為 1 號
   
   步驟 4: 按 Esc 關閉覆蓋層
   步驟 5: 按 Ctrl+Alt+Q，直接按 1
   預期: 立即跳轉到標記為 1 號的視窗
   ```

### 測試 2: 標籤模式 (Win+Y)

1. **準備**:
   - 開啟 3-5 個視窗
   - **重要**: 確保視窗**可見**且**未最小化**（標籤會顯示在視窗標題列上方）

2. **測試 - 直接跳轉**:
   ```
   步驟 1: 按 Win+Y
   預期: 每個可見視窗的標題列上方出現小標籤（60x48 像素）
         標籤顯示字母 A, S, D, F, G...
   
   步驟 2: 按字母鍵（例如 F）
   預期: 立即跳轉到對應視窗，標籤消失
         ⚠️ 不需要按 Enter，直接跳轉！
   ```

3. **測試 - 取消操作**:
   ```
   步驟 1: 按 Win+Y
   預期: 標籤出現
   
   步驟 2: 按 Esc
   預期: 標籤消失，停留在當前視窗
   ```

4. **測試 - 選擇後確認**（可選）:
   ```
   步驟 1: 按 Win+Y
   步驟 2: 按字母鍵（例如 D）
   預期: 立即跳轉（不會停留在選擇狀態）
   ```

## 🐛 常見問題

### Q1: Ctrl+Alt+Q 沒反應
**可能原因**: 
- 程式沒有執行（檢查系統匣圖示）
- 快捷鍵被其他程式佔用

**解決方法**:
1. 檢查系統匣是否有 Window Selector 圖示
2. 查看日誌: `%APPDATA%\window-selector\logs\`
3. 嘗試修改快捷鍵（編輯 config.toml）

### Q2: Win+Y 沒有顯示標籤
**可能原因**:
- 視窗被最小化（標籤只顯示在可見視窗上）
- 視窗被完全遮擋

**解決方法**:
1. 確保至少有一個視窗可見
2. 將視窗還原（不要最小化）
3. 檢查日誌確認標籤是否創建成功

### Q3: 標籤位置不對
**說明**: 標籤位置在啟動時計算，視窗移動後標籤不會跟隨

**解決方法**:
1. 按 Esc 關閉標籤
2. 重新按 Win+Y（標籤會重新計算位置）

### Q4: 按字母後沒有跳轉
**檢查**:
1. 確認按的字母對應到某個視窗（標籤上有顯示）
2. 確認視窗仍然存在（沒有被關閉）
3. 查看日誌是否有錯誤訊息
